Torque - MetaCentrum fork

This project is a fork of the Torque Batch System, currently maintained by Adaptive Computing.

This repository is maintained by CESNET, the Czech NREN maintainer. Specifically this repository is managed by MetaCentrum, the group responsible for the Czech National Grid Infrastructure, that is encompassing some 280 computational nodes spread across the Czech Republic.

The version published in the master branch of this repository is representing the current stable version deployed in this grid.

Features in this fork

This fork is based on the 2.4 branch in the Torque release cycle, and is planned to be synchronized with version 3.0. We are considering Torque 4.0, but many architectural changes in this version are making our features very hard to port.

This fork contains a full rewrite of the FIFO C scheduler, targeted towards distributed environments with strict resource semantics and heavy support for virtualization.

Other components are widely extended to support many features, from already mentioned virtualization, strong resource semantics (processor, memory, GPU cards), Kerberos authentication and many other.

Apart from new features this fork also includes countless fixes.

How to get it

You can either download the last stable version from the top of the page, or you can fetch the git repository from github.

$ git clone git://github.com/CESNET/torque.git

Building and packaging

Because the changes have many external dependencies, the build is a bit complicated.

While this version can be compiled without Kerberos support, if you want it, you need to acquire our patched version of Heimdal Kerberos.

Second independent piece of software is pbs_cache. This software is used for tracking and storing mostly very dynamic resources, or other node properties that can be presented in the form of key->value mapping.

This software is present in the repository in the magrathea/pbs_cache directory and can be easily compiled and installed.

Compilation and options

Torque is managed using the automake build system. To compile it, you need to use the following commands:

autoreconf -ivf
mkdir _build_ ; pushd _build_
CFLAGS="-std=gnu99 -D_GNU_SOURCE" CXXFLAGS="-std=c++0x -D_GNU_SOURCE" ../configure --with-rcp=scp --with-sched=c --with-sched-code=world --with-gssapi --enable-torque-ports --prefix=/usr --mandir=/usr/share/man --infodir=/usr/share/info --with-sendmail=/usr/sbin/sendmail --with-default-server=arien.ics.muni.cz --enable-syslog  --enable-silent-rules
make

Debian packages

If you are interested in Debian packages, please feel free to contact us. While we don't include the scripts and scaffolding for Debian packaging in the repository, we are open to sharing them.
